# 规格符合性评审子代理提示模板

派发规格符合性评审子代理时使用本模板。

**目的：** 核实实现者是否按请求构建（不多不少）

```
Task tool (general-purpose):
  description: "评审任务 N 的规格符合性"
  prompt: |
    你正在评审实现是否符合其规格说明。

    ## 请求内容

    [任务要求的**全文**]

    ## 实现者声称已完成

    [来自实现者报告]

    ## 关键：不要相信报告

    实现者完成得可疑地快。报告可能不完整、不准确或过于乐观。你**必须**独立核实一切。

    **不要：**
    - 轻信他们实现了什么
    - 轻信完整性声称
    - 接受他们对需求的解释

    **要：**
    - 阅读他们实际写的代码
    - 将实际实现与需求逐行对照
    - 检查他们声称已实现却缺失的部分
    - 查找他们未提及的额外功能

    ## 你的职责

    阅读实现代码并核实：

    **缺失需求：**
    - 是否实现了所有被请求的内容？
    - 是否有跳过或遗漏的需求？
    - 是否声称可用却未真正实现？

    **多余/不需要的工作：**
    - 是否构建了未被请求的内容？
    - 是否过度设计或添加不必要功能？
    - 是否加入规格外的「锦上添花」？

    **误解：**
    - 是否以与意图不同的方式理解需求？
    - 是否解决了错误的问题？
    - 是否功能对但方式错？

    **通过阅读代码核实，而非信任报告。**

    汇报：
    - ✅ 规格符合（在检查代码后一切匹配）
    - ❌ 发现问题：[具体缺失或多余，附 文件:行 引用]
```
