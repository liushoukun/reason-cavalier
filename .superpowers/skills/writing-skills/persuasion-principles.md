# 技能设计中的说服原则

## 概述

LLM 与人类一样会对说服原则产生反应。理解这种心理学有助于设计更有效的技能 —— 目的不是操纵，而是确保关键实践在压力下仍被遵循。

**研究基础：** Meincke 等（2025）在 N=28,000 段 AI 对话中测试 7 条说服原则；说服技巧使遵从率提高一倍以上（33% → 72%，p < .001）。

## 七条原则

### 1. 权威（Authority）
**是什么：** 对专业、资质或官方来源的服从。

**在技能中：** 祈使语气：「你必须」「绝不」「始终」；不可协商框架：「无例外」——减少决策疲劳与自我合理化。

**用于：** 纪律型技能（TDD、验证要求）、安全关键实践、既定最佳实践。

**示例：**
```markdown
✅ 先写实现再写测试？删掉。重来。无例外。
❌ 在可行时可以考虑先写测试。
```

### 2. 承诺（Commitment）
**是什么：** 与先前行为、陈述或公开宣示保持一致。

**在技能中：** 要求宣告使用技能；强制显式选择「选 A、B 或 C」；用 TodoWrite 跟踪清单。

**用于：** 确保技能真被遵循、多步流程、问责机制。

### 3. 稀缺（Scarcity）
**是什么：** 来自时限或有限可用性的紧迫感。

**在技能中：** 「再继续之前」等时限；「在 X 之后立即」等顺序依赖——对抗拖延。

### 4. 社会认同（Social Proof）
**是什么：** 与「他人做法」或常态一致。

**在技能中：** 「每次」「始终」等普遍模式；「无 Y 的 X = 失败」等失败模式——建立规范。

### 5. 同一（Unity）
**是什么：** 共享身份、「我们感」、ingroup。

**在技能中：** 「我们的代码库」「我们是同事」；共享目标「我们都想要质量」。

**用于：** 协作工作流、团队文化、非层级实践。

### 6. 互惠（Reciprocity）
**是什么：** 回报所得好处的义务。

**在技能中：** 少用 —— 易显操纵；技能中几乎不需要。

### 7. 喜好（Liking）
**是什么：** 更愿意与我们喜欢的人合作。

**在技能中：** **不要**用于合规 —— 与诚实反馈文化冲突，易谄媚。**纪律执行始终避免。**

## 按技能类型组合原则

| 技能类型 | 宜用 | 宜避 |
|----------|------|------|
| 纪律型 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 适度权威 + 同一 | 过重权威 |
| 协作型 | 同一 + 承诺 | 权威、喜好 |
| 参考型 | 仅清晰表达 | 全套说服 |

## 为何有效：心理学

**亮线规则减少合理化：**「你必须」消除「这是否例外？」的疑问；绝对措辞；显式反合理化表堵住具体漏洞。

**实现意图促成自动行为：** 清晰触发 + 必需行动 = 更易自动执行；「当 X 则 Y」比「一般做 Y」更有效。

**LLM 的类人性：** 训练数据含大量人类文本中的上述模式；权威语言常先于服从出现；承诺序列（陈述→行动）常被建模；社会认同（人人都做 X）建立规范。

## 伦理使用

**正当：** 确保关键实践被遵循；写有效文档；预防可预见失败。  
**不正当：** 为私利操纵；制造虚假紧迫；内疚式服从。

**检验：** 若用户完全理解该技巧，它是否仍服务于其真实利益？

## 速查

设计技能时自问：类型？要改变的行为？适用哪条原则（纪律型多为权威+承诺）？是否叠太多？是否伦理？
